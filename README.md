# Sass/JS Web制作スターターテンプレート

Webサイト制作、LPコーディング案件で即座に開発を始められるように、必要な環境設定とファイルを最適化したスターターテンプレートです。

Sass（SCSS）のコンパイルとJavaScriptのファイル読み込みが初期設定されており、環境構築の手間を省き、すぐにコーディングに集中できます。

## テンプレートの目的
- **案件の効率化:** 毎回行う環境構築（Sassやファイル構造）の時間をゼロにし、案件の受注・着手スピードを向上させる。
- **コード品質の担保:** プロ仕様の最適なファイル構成とHTMLのSEO/OGP設定を最初から適用する。
- **学習環境:** Sass、JavaScriptのDOM操作、モダンなHTML/CSSの記述練習に活用する。

## 含まれる主要ファイルと機能

| ファイル/フォルダ | 役割 | 機能 |
| :--- | :--- | :--- |
| `index.html` | HTML本体 | ビューポート、文字コード、SEO/OGPメタタグを最適化済み。 |
| `scss/` | Sassソース | SCSSファイルからCSSを生成するためのソースフォルダ。 |
| `css/style.css` | スタイルシート | SCSSからコンパイルされた最終的なCSSファイル。 |
| `js/script.js` | JavaScript | `defer` 属性で読み込みを最適化。DOM操作コードなどを記述する。 |
| `img/` | 画像 | サイト内で使用する画像を格納するフォルダ（空の状態）。 |
| `package.json` | 開発設定 | Sass（`sass`）の実行に必要な設定を記述。 |

## 🚀 開発環境の立ち上げ方

このテンプレートをダウンロード後、以下の手順で開発をすぐに開始できます。

### 1. 依存関係のインストール

ターミナルでフォルダに移動し、Sassの実行に必要なライブラリをインストールします。

```bash
npm install

2. Sassのコンパイル（監視開始）
以下のコマンドでSCSSファイルの変更を監視し、自動的に css/style.css に変換（コンパイル）を開始します。

Bash

npm run sass
Sassが起動したら、「Sass is watching for changes. Press Ctrl-C to stop.」と表示されます。

終了する際は Ctrl + C を押してください。
